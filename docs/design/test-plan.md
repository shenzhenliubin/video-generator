## Key Interactions to Verify

### Pipeline Flow
- **Watcher → Database**: New video detection saves metadata
- **Fetcher → Parser**: Subtitle download and parsing
- **Parser → Analyzer**: Cleaned text to LLM analysis
- **Analyzer → Writer**: Main points to rewritten script
- **Writer → Director**: Script to storyboard
- **Director → Artist**: Scene prompts to generated images
- **Director → Voice**: Narration to audio files
- **Artist + Voice → Renderer**: Images and audio to final video

### Provider Switching
- Switch between OpenAI/Anthropic LLM via config
- Switch between DALL-E/Stability image via config
- Switch between ElevenLabs/Local TTS via config
- Fallback to alternative provider on failure

### Checkpoint System
- Save state after each pipeline stage
- Resume from last checkpoint after failure
- Clean up stale checkpoints

## Edge Cases

### YouTube API
- Channel has no new videos
- Video has no subtitles available
- Subtitle format is unsupported
- API rate limit hit

### LLM Providers
- API key invalid or expired
- Rate limit exceeded
- Response timeout
- Malformed JSON response
- Content safety filter triggered

### Image Generation
- Prompt too long
- Content policy violation
- Generation failure
- Invalid image data returned

### TTS
- Text too long for single request
- Unsupported voice ID
- Audio format conversion failure

### Pipeline
- Concurrent video processing (resource contention)
- Disk space exhausted
- Database connection lost
- Partial failure (some stages succeed, others fail)

## Critical Paths

### End-to-End: Subtitle to Video
1. Watcher detects new video → saves to database
2. Fetcher downloads subtitle
3. Parser cleans and segments text
4. Analyzer extracts main points via LLM
5. Writer rewrites in target style via LLM
6. Director creates storyboard with scene descriptions
7. Artist generates images for each scene
8. Voice generates audio for each scene
9. Renderer combines images and audio into video
10. Video file saved to output directory

### Recovery Path
1. Pipeline fails at stage N
2. Checkpoint N-1 loaded
3. Stage N re-executed
4. Pipeline continues from N+1

### Provider Fallback Path
1. Primary provider API call fails
2. Fallback provider invoked
3. Processing continues with fallback result
4. Failure logged for monitoring
