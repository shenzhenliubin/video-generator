## Problem Statement

我想做知识类内容副业（YouTube/TikTok/抖音），但因为视频制作流程太复杂（找素材、写文案、剪辑、配音），我根本不开始做。

我需要一个工具来**降低高质量视频创作的门槛**——通过订阅我选择的 YouTube 博主，自动提取他们新视频的字幕，用 AI 重写文案，生成分镜、图片和语音，最终合成视频。

**核心假设:** 表达方式比观点本身更重要。我可以用更吸引人的方式重新包装别人已有的优质内容。

## Demand Evidence

- **当前状态:** 因为流程太麻烦，我根本不做视频
- **痛点确认:** 如果工具能自动化整个流程，我会开始做内容副业
- **目标:** 内容副业收入

## Status Quo

**现状:** 放弃不做

**替代方案成本:**
- 完全手工流程需要数小时到数天
- 现有工具（剪映、HeyGen 等）只能部分自动化，仍需大量手工操作
- 因为太麻烦，直接放弃

## Target User & Narrowest Wedge

**目标用户:** 我自己（以及类似想做内容副业但不想/不会做视频的人）

**最小切入点:**
- 只订阅我手工筛选的优质 YouTube 博主
- 自动化整个"字幕 → 视频"流水线
- 输出可直接发布的视频

## Constraints

- 技术栈未定（需要选择）
- 图像生成和语音合成的质量和成本是未知数
- YouTube API 限制和字幕格式多样性
- 分镜逻辑需要设计（如何将文案分割成视觉片段）
- 风格一致性（图片风格、语音风格需要统一）

## Premises

1. **表达优先于观点**——重新包装别人的内容可以通过更好的表达方式创造价值
2. **YouTube 博主已筛选**——我订阅的博主质量有保障，不需要额外的内容筛选机制
3. **自动化降低门槛**——如果流程足够自动化，我会开始做视频
4. **技术可行性**——现有的 LLM、图像生成、语音合成技术可以达到可接受的质量

## Approaches Considered

### Approach A: Minimum Viable Product (文案生成器)

只做 YouTube 字幕提取 + AI 重写，输出重写后的文案。

- **Effort:** S (小)
- **Risk:** 低
- **Pros:** 快速验证需求（2-3天），技术栈简单
- **Cons:** 不生成视频，需要手动使用其他工具
- **Reuses:** 无

### Approach B: Complete Automation Pipeline (推荐)

YouTube 订阅监控 → 字幕提取 → AI 观点分析 → 风格重写 → 分镜生成 → 风格一致图片生成 → 语音合成 → MoviePy 视频合成。

- **Effort:** XL (超大)
- **Risk:** 高（质量不可控）
- **Pros:** 完整的产品愿景，真正的"一键生成视频"
- **Cons:**
  - 每个环节都是技术债务
  - 图像和语音生成质量不可控
  - API 成本（DALL-E、ElevenLabs 等）
- **Reuses:** 无

### Approach C: Template-Driven Generator (横向思路)

预设 3-5 种流行视频风格模板，用户输入话题，系统按模板生成。

- **Effort:** M (中)
- **Risk:** 中
- **Pros:** 模板可控，质量有保障
- **Cons:** 需要主动搜索内容，比订阅模式更复杂

**用户反馈:** 方案 C 需要平台化和主动内容搜索，工作量比方案 B 更大。方案 B 的输入（订阅的博主）是确定的，不需要额外筛选。

## Recommended Approach

**选择方案 B: Complete Automation Pipeline**

**理由:**
1. 用户已经通过订阅做了内容筛选，输入质量有保障
2. 这是用户的完整产品愿景，分阶段实现会削弱价值
3. 确定的流水线工程比主动内容搜索更简单

**风险缓解:**
- 分阶段实现：先验证字幕提取和 AI 重写质量，再加图片/语音/视频
- 使用可配置的提示词来控制生成质量
- 提供人工编辑入口（在生成视频前可以修改文案/分镜）

## Open Questions

1. **分镜逻辑:** 如何将重写后的文案分割成适合视觉呈现的片段？
2. **风格一致性:** 如何保证生成的图片和语音风格统一？
3. **字幕格式:** YouTube 字幕格式多样（自动生成、手动字幕、多语言），如何处理？
4. **API 成本:** 图像和语音生成的单次成本是多少？可接受的预算是多少？
5. **法律问题:** 重新包装别人内容的版权边界在哪里？

## Success Criteria

1. 能成功订阅 YouTube 频道并检测新视频
2. 能正确提取和解析字幕
3. AI 重写后的文案质量可接受（不奇怪、有逻辑）
4. 生成的视频可发布到平台（质量过关）
5. 端到端流程可以在 1 小时内完成（从检测到视频生成）

## Dependencies

- YouTube Data API（订阅和视频信息）
- 字幕下载工具（如 `youtube-transcript-api`）
- LLM API（Claude/GPT 用于文案重写）
- 图像生成 API（DALL-E/Midjourney/Stability AI）
- 语音合成 API（ElevenLabs/Azure TTS/Google TTS）
- MoviePy（视频合成）
